dockerに関するコマンド集
docker-compose.ymlを使用した、コンテナの起動
・docker compose up -d --build 
コンテナの確認
docker container ls(;sとうつとパソコンがシャットダウンするから注意！)
・イメージが作られているかの確認
docker images
実際に仮想環境に入る方法
・docker compose exec 【コンテナ名】 bash
Ex）docker compose exec python3 bash(デフォルトであげたファイルなら、このコマンドをそのままパクればよい)
名称の設定はdocker-compose.ymlの中のcontainer_name
・pythonfileがあるoptフォルダに移動
cd opt
・pythonfileの実行
python [pythonファイル名.py]
Ex)python test2.py
dockerから出る方法
・control+Z

仮想環境の消し方
1.コンテナの消し方
・docker rmi 【コンテナ名】 -f
Ex)docker rmi python3 -f
2.イメージの消し方
・docker rm 【イメージ名】
Ex)docker rm step1-python3
↑これについては、docker imagesより、リポジトリ名を参照


その他
コンテナが存在しているかの確認
docker ps -a
コンテナが起動しているかの確認
docker ps
もし動いていないなら、コンテナを開始する方法
docker start 【CONTAINER ID】
Ex)docker start 56fefd7c85c0
エラー対応方法
もし、コンテナが消せない場合
docker stop 【CONTAINER ID】
Ex)docke stop 6426d7038245
その後、コンテナを削除
docker rmi 6426d7038245
